# 肠音三分类分析报告

## 1. 数据概况

### 1.1 数据来源

- 数据目录：`data_changyin/`
- 格式：628 个 WAV 文件 + 628 个对应 TXT 标注文件
- 采样率：44100 Hz（处理时重采样至 16000 Hz）
- 标注格式：`start_time<TAB>end_time<TAB>[label]`
  - 第三列为空时默认为 `n`

### 1.2 类别分布

| 类别 | 事件数 | 占比 | 总时长 | 平均时长 |
|------|--------|------|--------|----------|
| n (正常) | 9,590 | 97.9% | ~222 min | ~1.4s |
| g (异常G) | 179 | 1.8% | ~9 min | ~3.0s |
| h (异常H) | 24 | 0.2% | ~17s | ~0.7s |

**核心矛盾**：h 只有 24 个样本，n:g:h 采样比为 9590:179:24（约 400:7:1），极端不平衡。

## 2. 物理特征分析

### 2.1 声学特征统计

基于实际音频片段的频谱分析（N_FFT=2048, HOP=512, SR=16000）：

| 特征 | n (均值) | g (均值) | h (均值) | h/n 比值 | 区分力 |
|------|----------|----------|----------|----------|--------|
| 频谱质心 (Hz) | 272 | 510 | **300** | 1.1x | ★★☆ |
| 频谱带宽 (Hz) | 202 | 954 | **201** | 1.0x | ★☆☆ |
| RMS 能量 | 1,626 | 2,197 | **9,620** | **5.9x** | ★★★★★ |
| 峰值因子 (crest) | 11.1 | — | **5.2** | 0.47x | ★★★★ |
| 过零率 (ZCR) | 0.046 | 0.024 | **0.036** | 0.78x | ★★★ |
| 时长 (s) | 1.28 | 8.80 | **0.71** | 0.55x | ★★☆ |

### 2.2 频段能量分布

| 频段 | h | n | g | 关键差异 |
|------|---|---|---|----------|
| 0-200Hz | 16.9% | 26.5% | — | h 低频占比更低 |
| **200-500Hz** | **75.3%** | 69.7% | — | **h 能量高度集中于此** |
| 500-1kHz | 5.9% | 2.0% | — | h=2.9× n |
| 1k-4kHz | 1.8% | 1.8% | — | 无差异 |
| >4kHz | ~0% | ~0% | — | 肠音天生无高频 |

### 2.3 h 类内部结构

24 个 h 样本按 RMS 能量可分为两个子类：

| 子类 | 数量 | RMS 范围 | 质心范围 | 时长范围 | 判读 |
|------|------|----------|----------|----------|------|
| **高能型** | 17个 | 8,000–17,000 | 209–393 Hz | 0.18–1.14s | h 的主体模式 |
| **低能型** | 7个 | 1,400–4,300 | 209–472 Hz | 0.86–2.00s | 可能为不同病理程度或采集距离差异 |

### 2.4 h 类物理特征模板

```
频谱质心:      300 ± 61 Hz      [180, 480]
频谱带宽:      201 ± 80 Hz      [60, 450]
200-500Hz能量:  75%             [60%, 100%]
过零率:        0.036 ± 0.009   [0.015, 0.060]
峰值因子:      5.2 ± 4.1       [1.5, 20.0]
>4kHz能量:     ~0%             [0, 10%]
```

### 2.5 关键结论

**h 的物理本质**：低频窄带高能量脉冲。200-500Hz 能量占 75%，RMS 是 n 的 6 倍。任何加性噪声或背景混合都会破坏这个极其纯净的频谱结构。

**g 与 n 的差异微弱**：质心只差约 238 Hz，带宽和能量分布有大量重叠。这是分类的主要难点。

## 3. 数据增强验证

### 3.1 h 类约束增强

经过验证，只有**频谱结构保持型**增强才能让 h 增强样本保持物理特征在原始分布内：

| 增强方法 | 约束通过率 | 安全? |
|----------|:---:|:---:|
| 时间拉伸 (0.88–1.12×) | 92–100% | ✅ |
| 音调偏移 (±150 cents) | 83–100% | ✅ |
| 增益扰动 (±6 dB) | 88–92% | ✅ |
| 相位随机化 (30%) | 100% | ✅ |
| 低频 EQ (±3 dB, 200–500Hz) | 79–92% | ✅ |
| 加性高斯噪声 (SNR 8–20dB) | **0%** | ❌ 破坏窄带结构 |
| 背景混合 (n类, SNR 2–10dB) | **17%** | ❌ 破坏纯净度 |
| 组合 (stretch+pitch+noise+mix) | **0%** | ❌ |

### 3.2 最终增强方案

```
h: 24 → 990 (×41), 约束通过率 99.9%
g: 179 → 1611 (×9), 自由增强
n: 9590 → 4000, 下采样
```

h 增强使用 8 种组合：timestretch(7) + pitchshift(6) + gain(5) + phase(4) + low_eq(3) + stretch+pitch(5) + stretch+gain(5) + pitch+gain(4) = 39 种/样本

## 4. BEATs 特征分离度验证

### 4.1 实验设置

- BEATs 模型：BEATs iter3 (AudioSet-2M 预训练, 90M 参数, 冻结)
- 输入：原始波形, 16kHz, 3秒定长（短片段补零, 长片段居中截取）
- 特征：Mean-pool over time → 768 维
- 分类器：Logistic Regression (C=1.0, class_weight='balanced')
- 采样：n=2000, g=179, h=24；分层 70/30 拆分

### 4.2 定量结果

**逻辑回归分类（BEATs 特征的理论上限）：**

| 类别 | Precision | Recall | F1-score | Support |
|------|-----------|--------|----------|---------|
| n | 0.946 | **0.927** | 0.936 | 600 |
| g | 0.382 | **0.482** | 0.426 | 54 |
| h | 0.600 | **0.429** | 0.500 | 7 |
| **Macro Avg** | 0.643 | 0.612 | 0.621 | 661 |
| **Weighted Avg** | 0.896 | 0.885 | 0.890 | 661 |

**混淆矩阵：**

```
         预测n  预测g  预测h
真实n     556    42     2       n→g 误判: 7.0% (42/600)
真实g      28    26     0       g→n 误判: 51.9% (28/54), g→h: 0%
真实h       4     0     3       h→n 误判: 57.1% (4/7),   h→g: 0%
```

**类间距离（标准化欧氏距离）：**

| | n | g | h |
|---|-----|-----|-----|
| n | 0 | 16.9 | 15.9 |
| g | 16.9 | 0 | **28.0** |
| h | 15.9 | **28.0** | 0 |

**Silhouette Score**: -0.097（负值，说明样本更接近其他类中心而非自身类中心）

**Fisher 判别比 (top-10 维度)**：

| 维度 | Fisher Ratio |
|------|:---:|
| dim_0378 | 0.1174 |
| dim_0606 | 0.0894 |
| dim_0747 | 0.0888 |
| dim_0081 | 0.0851 |
| dim_0283 | 0.0841 |
| dim_0542 | 0.0823 |
| dim_0228 | 0.0817 |
| dim_0481 | 0.0816 |
| dim_0430 | 0.0811 |
| dim_0071 | 0.0799 |

全部 < 0.12，说明没有单一维度能干净分离三类——分离信息分布在整个 768 维空间中。

### 4.3 关键发现

**发现 1：g 和 h 在 BEATs 空间里分离极好**

距离 28.0（远超 n↔g 的 16.9），混淆矩阵中 g↔h 互混为零。BEATs 冻结状态下就能完美区分 g 和 h。

**发现 2：瓶颈在 n↔g 区分**

g 召回率仅 48%（52% 的 g 被误判为 n），42 个 n（7%）被误判为 g。n 与 g 的 BEATs embedding 有大量重叠。

**发现 3：h 召回率低但信号不脏**

h 的 4 个错误全部误判为 n，没有一个误判为 g。低召回率部分原因是测试集只有 7 个 h 样本（统计不稳定），但也反映 h 和 n 的 embedding 距离（15.9）与 n↔g 距离（16.9）相近。

**发现 4：Fisher 比极低 → 无单维度分离**

没有任何特征的 Fisher 比超过 0.12。分类信息在 768 维空间中分散编码，需要高维非线性分类器（如投影头 + 对比学习）才能有效利用。

## 5. 综合结论

### 5.1 三阶段方案

```
Stage 1: n vs (g+h) 二分类
  方法: BEATs + SupMin 对比学习
  预期: n recall ~93%, rare recall ~50%
  瓶颈: g 和 n 的频谱差异太小 (质心差 ~238Hz)

Stage 2: g vs h 分类
  方法: 物理特征 (7维) + 轻量分类器 (SVM/规则)
  预期: 几乎完美分离 (RMS 差 4.4倍, 零互混)
  关键特征: RMS, crest, lo_energy_ratio, centroid

Stage 3 (可选): n vs g 精细区分
  方法: BEATs embedding + 物理特征 concat (768+7=775维)
         → projection → 对比学习
  预期: g recall 可提升至 65-75%
```

### 5.2 当前方案的局限性

| 问题 | 根因 | 缓解措施 |
|------|------|----------|
| g recall 上限 ~48% | BEATs 预训练域 (AudioSet) 与肠音的频谱差异不匹配；n 和 g 的物理差异 (质心差 53Hz in finer measurement) 在 768 维 embedding 中被淹没 | 解冻 BEATs 最后 2 层；或 concat 物理特征 |
| h 增强多样性有限 | 24 个原始样本全部来自相似采集条件 | 物理约束增强已确保 99.9% 的增强样本在 h 分布内 |
| BEATs 与 AudioSet 域偏移 | AudioSet 是环境音（狗叫、汽车、乐器），肠音是体内生物声 | 微调或换 PaSST/AST（已用于声音事件检测的预训练模型）|

### 5.3 推荐优先级

1. **Stage 1 + Stage 2 组合先跑通**（当前 TTC_changyin 代码已就绪）：BEATs + SupMin 做 n vs rare，物理特征做 g vs h
2. **若 Stage 1 的 rare recall 不满足需求**：尝试 BEATs + 物理特征 concat（775 维输入，修改 projection head 输入维度即可）
3. **若仍不满足**：换 PaSST 或 HTS-AT（项目已有 HTS-Audio-Transformer 代码）作为音频编码器
